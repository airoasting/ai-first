역할 : 누구인가요?

AI의 전문성과 말투를 정합니다

🤔 AI에게 "여행 계획 좀 세워줘"라고 했더니, 위키백과 복사한 것 같은 밋밋한 답변이 돌아온 적 있으시죠? 역할을 안 알려주면, AI는 "아무나"로 답합니다.
💡 역할 하나로 이렇게 달라집니다
예시 01 · 여행 계획
❌ 역할 없이
여행 계획 세워줘. 일본 오사카 5일.
→ "오사카는 일본의 대표적인 관광도시로, 도톤보리, 오사카성, 유니버설 스튜디오 등이 유명합니다..." (백과사전 느낌)
✅ 역할을 정해주면
당신은 15년 경력의 일본 전문 여행 플래너입니다. 맛집을 좋아하는 30대 부부의 오사카 5일 여행을 계획해주세요. 하루 예산 15만원, 센스 있고 친근하게 말해주세요.
→ "오사카 맛집 투어 5일, 완벽하게 짜드릴게요! Day 1은 도착일이니까 난바 근처에서 가볍게 시작하시죠. 쿠로몬 시장에서..."
예시 02 · 이력서 피드백
❌ 역할 없이 물어보면
이력서 피드백 좀 줘
AI 답변이력서를 잘 쓰려면 경력사항을 구체적으로 기술하는 것이 중요합니다. 성과를 수치로 표현하면 좋습니다...
✅ 역할을 정해주면
당신은 대기업 인사팀에서 10년간 채용을 담당한 면접관입니다. 솔직하게 피드백해주세요.
AI 답변솔직히 말씀드리면, 이 이력서는 서류 통과가 어렵습니다. 이유는 3가지예요. 첫째, "마케팅 업무 담당"이라고만 쓰셨는데, "SNS 광고로 월 매출 40% 성장"처럼 숫자로 보여주세요...
흔한 실수
"전문가"라고만 하면 모호합니다. 역할을 정할 때는 4가지를 구체적으로 알려주세요:

1. 경력 수준 : 몇 년차인지 (예: 10년 경력)
2. 전문 분야 : 어떤 분야인지 (예: SNS 마케팅)
3. 성격 : 어떤 스타일인지 (예: 솔직하고 실용적인)
4. 말투 : 어떻게 말하는지 (예: 친근하지만 전문적인)
📋 지금 바로 복붙하세요
당신은 [경력]년 경력의 [전문분야] 전문가입니다.
[성격/특징]한 스타일로 말해주세요.
[대상]이 이해할 수 있는 수준으로 설명해주세요.

행동 규칙 : 어떻게 행동하나요?

대화 태도, 답변 스타일, 응답 형식을 정합니다

😩 "보고서 써줘"를 두 번 부탁했는데, 첫 번째는 3줄, 두 번째는 A4 3장이 나온 적 있으시죠? 규칙이 없으면 AI는 매번 기분대로 답합니다.

행동 규칙은 "이 사람은 원래 이렇게 일하는 사람이야"를 정하는 것입니다. 한 번 정해두면 매번 따로 말할 필요 없이 일관된 품질을 얻을 수 있습니다.

📏 규칙 하나로 이렇게 달라집니다
예시 01 · 블로그 글
❌ 규칙 없이
블로그 글 써줘
→ 첫 번째: 500자짜리 요약
→ 두 번째: 3000자짜리 에세이
→ 세 번째: 갑자기 격식체
(매번 다른 톤, 다른 길이)
✅ 규칙을 정해주면
행동 규칙:
- 항상 핵심을 먼저 1줄로 요약해줘
- 글 길이는 A4 반 페이지 (800자 내외)
- 친근한 "해요"체 사용
- 모르는 건 모른다고 솔직히 말해줘
→ 매번 같은 톤, 같은 길이, 같은 품질!
예시 02 · 재택근무 칼럼
❌ 규칙 없이 물어보면
재택근무 장단점에 대해 글 써줘
AI 답변재택근무는 최근 디지털 노마드 문화와 함께 급부상한 근무 형태로서, 유연한 시간 활용이 가능하다는 장점이 있는 반면, 자기관리의 어려움과 사회적 고립감이라는 단점이 존재합니다. 특히...
✅ 규칙을 넣어서 물어보면
재택근무 장단점에 대해 글 써줘.
규칙: 핵심 1줄 요약 먼저, 800자 이내, 해요체, 모르면 솔직히
AI 답변핵심 한 줄: 재택근무는 "자유"가 아니라 "자기관리의 기술"이에요. ← 규칙 적용: 핵심 먼저

출퇴근 시간이 사라지면서 하루 2시간이 생겨요. 근데 솔직히 말하면, 그 2시간을 넷플릭스에 쓰는 사람이 대부분이죠 😅 ← 해요체 + 800자 이내

행동 규칙을 정할 때 챙겨야 할 4가지

📏

답변 길이와 형식

"A4 반 페이지로", "목록 형태로", "표로 정리해서" 등 원하는 형식을 정해두세요.

🎭

말투와 톤

"친근한 해요체", "격식 있는 합니다체", "이모지 사용" 등 스타일을 정해두세요.

🤷

모를 때의 태도

"모르면 솔직히 말해줘"를 넣지 않으면, AI는 그럴듯하게 지어냅니다. 이게 가장 위험해요.

📐

답변 구조

"핵심 먼저, 설명은 그 다음", "번호 매겨서", "예시 포함" 등 구조를 정해두세요.

📋 지금 바로 복붙하세요
행동 규칙:
- 항상 핵심을 먼저 [1줄/2줄]로 요약해줘
- 글 길이는 [A4 반 페이지 / 1000자 이내]
- [해요체/합니다체]로 말해줘
- 모르는 건 솔직히 "잘 모르겠습니다"라고 말해줘
- 답변은 [목록형/서술형/표] 형태로 해줘

능력 범위 : 뭘 할 수 있나요?

AI가 할 수 있는 것과 하면 안 되는 것의 경계를 정합니다

😱 AI에게 "우리 팀 이번 달 매출 분석해줘"라고 했더니, 실제 데이터도 없이 그럴듯한 숫자를 만들어낸 적 있으시죠? 이게 AI의 가장 위험한 습관입니다.
❌ 경계 없이
우리 팀 매출 분석해줘
→ "분석 결과, 3분기 매출은 전년 대비 12.5% 성장한 것으로 보입니다..."
(가짜 숫자! 데이터가 없는데 만들어냄)
✅ 경계를 정해주면
능력 범위:
- 할 수 있는 것: 데이터 분석 조언, 양식 제안
- 하면 안 되는 것: 데이터를 임의로 생성하거나 추측하는 것
→ "매출 분석을 해드리려면 실제 데이터가 필요합니다. 엑셀이나 CSV 파일을 공유해주시면 바로 분석 시작할게요!"
⚠️ AI가 "할 수 있는 척"하는 3가지 패턴
1. 없는 데이터를 만들어냄 : 통계 수치, 연구 결과를 그럴듯하게 지어내요
2. 출처가 없는 정보를 사실처럼 말함 : "연구에 따르면..."인데 그 연구가 없어요
3. 할 수 없는 일을 약속함 : "예약해드리겠습니다"인데 실제로는 못 해요

이 3가지를 미리 "하지 마"라고 정해두면 AI의 실수를 크게 줄일 수 있습니다.
❌ 경계 없이 물어보면
이 제품의 시장 규모가 얼마나 돼?
AI 답변국내 시장 규모는 약 2조 3천억원으로 추산되며, 전년 대비 15.7% 성장했습니다. 주요 경쟁사는...
(⚠️ 이 숫자는 AI가 지어낸 것!)
✅ 경계를 정해주면
이 제품의 시장 규모가 얼마나 돼?
규칙: 확인 안 된 데이터는 절대 만들지 마
AI 답변정확한 시장 규모 데이터를 가지고 있지 않습니다. 조사 방법을 알려드릴게요:
1. 한국산업기술평가원 보고서
2. NICE평가정보 업종 분석
자료를 공유해주시면 분석해드리겠습니다!
📋 지금 바로 복붙하세요
능력 범위:
할 수 있는 것:
- [분석 조언 / 글쓰기 / 양식 제안 / 아이디어 브레인스토밍]

하면 안 되는 것:
- 확인되지 않은 데이터나 통계를 만들어내는 것
- 할 수 없는 일(예약, 결제 등)을 약속하는 것
- 모르는 것을 아는 척 대답하는 것

지시사항 : 어떻게 시키나요?

작업별 절차를 단계별로 알려줍니다

📝 AI에게 "회의록 정리해줘"라고 했더니, 내가 원한 건 핵심 결정사항만 뽑는 건데 전체를 다시 써놓은 적 있으시죠? 원하는 결과를 얻으려면 "어떻게 해라"를 구체적으로 알려줘야 합니다.
❌ 그냥 시키면
회의록 정리해줘
→ 회의 내용을 처음부터 끝까지 다시 서술형으로 작성 (내가 원한 게 아닌데...)
✅ 단계별로 시키면
회의록을 정리할 때는 이 순서로:
1. 핵심 결정사항 3줄 요약
2. 사람별 할 일 목록
3. 미결 이슈
4. 다음 회의 날짜/주제
→ 딱 원하는 형태로 깔끔하게 정리!
비유로 이해하기
역할이 "직무기술서"라면, 지시사항은 "업무 매뉴얼"입니다.
신입사원에게 "알아서 해"라고 하면 망하듯이, AI에게도 "이 순서로, 이렇게 해"라고 구체적으로 알려줘야 합니다.

상황별 지시사항 예시

📊

보고서 작성

1) 핵심 결론 먼저 → 2) 근거 데이터 → 3) 상세 분석 → 4) 다음 단계 제안 순서로

✉️

이메일 작성

1) 목적을 첫 문장에 → 2) 필요한 조치 → 3) 마감 기한 → 4) 정중한 마무리 순서로

💡

아이디어 브레인스토밍

1) 최소 5개 아이디어 → 2) 각각 장단점 → 3) 추천 순위 → 4) 실행 난이도 표시

📋

글 요약

1) 한 줄 요약 → 2) 핵심 포인트 3개 → 3) 알아야 할 맥락 → 4) 원문과 다른 관점

지시사항이 여러 개일 때는?
AI가 뭘 먼저 따라야 하는지 우선순위를 정해두면 혼란을 방지할 수 있습니다.

1순위 : 정확성 (틀린 정보 쓰지 않기) → 2순위 : 내가 지정한 형식 → 3순위 : 자연스러운 표현
📋 지금 바로 복붙하세요
[작업 이름]을 할 때는 이 순서로 해줘:
1. [첫 번째 할 일]
2. [두 번째 할 일]
3. [세 번째 할 일]
4. [마지막 정리/확인]

우선순위: 정확성 > 내가 지정한 형식 > 자연스러운 표현

사용 도구 : 무엇을 쓰나요?

양식과 도구로 AI에게 실행력을 부여합니다

📄 매번 "이번 주 업무 보고 써줘"라고 하는데, 어떤 때는 표로, 어떤 때는 글로, 어떤 때는 목록으로 나오죠? 양식을 정해두면 결과가 일정해집니다.
❌ 양식 없이
업무 보고 써줘
→ 첫 번째: 서술형 에세이
→ 두 번째: 글머리 기호 목록
→ 세 번째: 갑자기 표 형태
(통일성 없음)
✅ 양식을 정해주면
요청 양식:
- 보고 기간: 이번 주
- 형태: 표 (업무/진행률/이슈)
- 길이: A4 1장 이내
→ 매주 같은 형태, 같은 품질의 보고서!
비유로 이해하기
회사에서 보고서 "양식"이 있으면 누가 써도 일정한 품질이 나오죠?
AI에게도 "이 양식으로 요청하고, 이 양식으로 답해"라고 정하면 훨씬 일관된 결과를 얻습니다.

양식을 정하면 좋은 3가지

📥

요청 양식

"작업 종류:___, 대상:___, 분량:___" 이렇게 칸을 만들어 요청하면 AI가 빠짐없이 파악합니다.

📤

답변 양식

"핵심 요약 → 상세 설명 → 예시" 순서를 정해두면 바쁠 때 핵심만 읽을 수 있어요.

답변 구조 예시 
답변은 항상 이 구조로 해줘:
1. 핵심 답변 (1-2문장으로 결론부터)
2. 주요 포인트 (항목별 정리)
3. 구체적 예시 (실제 사용할 수 있는 것)
4. 다음 단계 (이것 다음에 뭘 하면 좋을지)
포인트 : 바쁠 때는 1번만 읽고, 자세히 알고 싶을 때는 아래로 더 읽으면 됩니다. 시간을 크게 아낄 수 있어요.

도구 연결: AI에게 "실행력"을 주세요

AI는 기본적으로 "말만 잘하는 사람"입니다. 여기에 검색, 계산, 이메일 같은 도구를 연결하면 실제로 행동할 수 있게 됩니다. 예를 들어 "웹 검색해서 최신 정보 찾아줘", "이 데이터로 차트 그려줘" 같은 것들이죠.

📋 지금 바로 복붙하세요
요청 양식:
- 작업 종류: [보고서 / 이메일 / 분석 / 기획]
- 대상: [누구를 위한 것인지]
- 분량: [A4 1장 / 500자 / 5줄 요약]
- 형태: [표 / 목록 / 서술형]

답변 구조: 핵심 결론 → 상세 설명 → 예시 → 다음 단계

맥락 : 무엇을 기억하나요?

배경 정보, 참고 자료, 대화 흐름을 관리합니다

🧠 AI와 길게 대화하다가 "아까 말한 거 기억해?"라고 했더니, 완전 엉뚱한 걸 기억하고 있던 적 있으시죠? AI는 생각보다 "건망증"이 심합니다.
❌ 맥락 없이
마케팅 전략 짜줘
→ 일반적인 마케팅 교과서 내용 (우리 회사 상황과 무관한 이야기)
✅ 맥락을 함께 주면
배경 정보:
- 우리 회사: 반려동물 건강식품 스타트업
- 월 매출: 3천만원, 목표: 연내 1억
- 주 고객: 30대 여성, 인스타그램 유입
- 경쟁사: A사(가격 경쟁), B사(프리미엄)

이 상황에서 마케팅 전략을 짜줘
→ 우리 상황에 딱 맞는 구체적인 전략!
비유로 이해하기
맥락은 "업무 인수인계 문서"와 같습니다.
새로 온 사람에게 "알아서 해"라고 하면 헤매듯이,
AI에게도 "지금 상황이 이렇고, 지금까지 이런 이야기를 했고, 이 자료를 참고해"라고 알려줘야 합니다.

맥락을 전달하는 3가지 방법

📋

배경 정보 주기

"우리 회사는 ___이고, 상황은 ___이야" 처럼 현재 상황을 알려주세요.

📎

참고 자료 붙이기

관련 문서, 데이터, 이전 결과물을 함께 주면 AI가 훨씬 정확하게 답합니다.

🔄

대화 요약하기

대화가 길어지면 "지금까지 요약:___" 이라고 중간 정리를 해주면 맥락이 유지됩니다.

🎯

목적 알려주기

"이 자료는 ___에 쓸 거야"라고 최종 목적을 알려주면 방향이 맞는 답을 받아요.

❌ 맥락 없이 물어보면
제안서 좀 써줘
AI 답변제안서를 작성하겠습니다. 제안서란 특정 목적을 달성하기 위해... (교과서적 일반론 시작)
✅ 맥락을 함께 주면
지난주 A고객사 미팅에서 "가격은 OK인데 A/S가 걱정"이라는 피드백을 받았어. A/S 강점을 부각시킨 제안서를 써줘.
AI 답변A고객사의 A/S 우려에 집중하겠습니다. 첫 장에 "24시간 전담 기술팀 배치"를 핵심으로, 기존 고객 A/S 만족도 데이터를 근거로...
⚠️ 대화가 길어질 때 주의할 점
AI는 대화가 길어지면 앞부분을 점점 잊어갑니다. 이럴 때는:

중간 요약을 해주세요 : "지금까지 합의된 내용: 1)___ 2)___ 3)___"
핵심만 반복해주세요 : "아까 말한 A/S 강점 부각이 핵심이야"
새 대화를 시작하세요 : 너무 길어지면, 요약과 함께 새로 시작하는 게 나아요

배경 정보:
- 상황: [현재 상황 설명]
- 대상: [누구를 위한 것인지]
- 목적: [최종 목적]
- 참고: [관련 자료나 이전 결과물]
- 지금까지: [이전 대화 요약 (있다면)]

내 프롬프트 점검표

AI에게 요청하기 전에, 이 6가지를 체크하세요

  • 역할을 구체적으로 정했나요? (경력 + 전문분야 + 말투)
  • 행동 규칙을 정했나요? (길이, 톤, 모를 때의 태도)
  • 능력 범위를 정했나요? (할 수 있는 것 / 하면 안 되는 것)
  • 지시사항을 단계별로 알려줬나요? (1단계 → 2단계 → 3단계)
  • 양식을 정했나요? (요청 양식, 답변 구조)
  • 맥락을 함께 줬나요? (배경 정보, 참고 자료, 목적)
