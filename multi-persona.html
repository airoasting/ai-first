AI 토론 설계법 (멀티 페르소나)

AI 한 명에게 묻지 말고, AI 팀을 구성해서 토론시키세요

대화만으로 퀄리티 2배 · 5분 소요 · 중급~고급
AI 활용 심화 시리즈 11가지 습관 멀티 페르소나 환각 줄이기

같은 질문, 다른 퀄리티

팀장 + 팀원 구성만 추가하면 답변의 깊이가 달라집니다

Before — 혼자 물어보기
"20대 직장인 대상 AI 생산성 앱을 런칭하려고 해. 마케팅 전략을 짜줘."
결과: 한 관점의 일반적 전략 나열
• SNS 마케팅, 인플루언서 협업, 콘텐츠 마케팅...
• 나름 괜찮지만, 누가 써도 나올 수 있는 답변
• 비용·고객·데이터 검증 없음
After — 팀 토론
"총 4명의 팀을 구성해줘. 마케팅 디렉터가 팀장이고, 데이터 분석가, 고객 전문가, 재무 담당이 팀원이야. 마케팅 디렉터가 각 팀원과 3차례 비판적으로 토론한 후, 최종 결론을 도출해줘."
결과: 3라운드 비판적 토론을 거친 전략
• 1라운드: 각 팀원의 초기 주장
• 2라운드: 팀장이 약점 지적 → 팀원이 보완
• 3라운드: 반박 & 재반박으로 논리 완성
• 검증된 최종 결론 도출

왜 효과적인가?

한 사람의 관점 vs 여러 전문가의 관점

🔍
다각도 검증
각 전문가가 자기 관점에서 빈틈을 찾아냅니다
💬
순차적 토론
앞사람 의견을 듣고 반박·보완하며 깊어집니다
✅
팀장 종합
토론 결과를 종합해 실행 가능한 결론을 냅니다

Anthropic 연구가 증명한 원리

왜 "역할을 나눠라"고 하면 AI가 진짜 다르게 답할까?

🎭

AI는 "배우"처럼 작동합니다

Anthropic의 Persona Selection Model(PSM) 연구에 따르면, Claude 같은 AI는 학습 과정에서 수많은 글을 읽으며 다양한 캐릭터를 연기하는 법을 배웁니다. 마치 배우가 여러 역할을 소화할 수 있는 것처럼요.

📚
학습 = 캐릭터 습득
수많은 텍스트를 읽으며 다양한 전문가의 사고방식을 학습
🎯
역할 부여 = 캐릭터 전환
"데이터 분석가처럼 봐줘"라고 하면 그 캐릭터의 사고방식으로 전환
🧠
실제로 다른 관점
단순 흉내가 아니라 AI 내부에서 실제로 다른 패턴이 활성화

그래서 멀티 페르소나가 효과적입니다. AI에게 서로 다른 역할을 부여하면, 각 역할마다 실제로 다른 전문가의 관점이 활성화됩니다. 한 명에게 물어보면 한 가지 관점만 나오지만, 팀을 구성하면 여러 관점이 충돌하며 더 깊은 답변이 나오는 이유입니다.

📎 출처: Anthropic, "The Persona Selection Model" (2026)

팀 구성하는 법

팀장 1명 + 팀원 2~4명이 가장 효과적입니다

👥

예시: 마케팅 전략 수립팀

팀장
마케팅 디렉터
최종 의사결정, 토론 종합 및 결론 도출
반드시 1명
팀원 1
데이터 분석가
수치 근거 제시, 시장 데이터 기반 의견
팀원 2
고객 경험 전문가
사용자 관점, 실제 고객 반응 예측
팀원 3
재무 담당
비용 타당성, ROI 분석, 예산 현실성 검토
팀 구성 팁: 팀원은 서로 다른 관점을 가진 역할로 구성하세요. 같은 방향만 바라보는 팀은 토론이 아니라 동의가 됩니다. "비판자" 역할을 하나 넣으면 퀄리티가 확 올라갑니다.

복사해서 바로 쓰는 프롬프트

아래 템플릿을 복사하고, 빨간 부분만 내 상황에 맞게 바꾸세요

프롬프트 템플릿
지금부터 팀 회의를 시작합니다. ## 논의 주제 [여기에 논의할 주제를 적으세요] ## 팀 구성 (총 4명) 팀장: [역할명] — 각 팀원과 비판적으로 토론하고, 최종 결론을 도출하는 역할 팀원 1: [역할명] — [이 역할의 관점/전문 분야] 팀원 2: [역할명] — [이 역할의 관점/전문 분야] 팀원 3: [역할명] — [이 역할의 관점/전문 분야] ## 진행 규칙 1. 각 팀원이 자기 관점에서 의견을 먼저 제시합니다 2. 팀장이 각 팀원과 1:1로 3차례씩 비판적 토론을 진행합니다 — 1차: 팀장이 약점·빈틈을 지적 → 팀원이 반박·보완 — 2차: 팀장이 재반박 → 팀원이 근거를 추가해 방어 — 3차: 양측이 합의점 도출 또는 쟁점 정리 3. 모든 팀원과의 토론이 끝나면, 팀장이 논의 내용을 종합하여 최종 결론을 도출합니다 ## 출력 형식 — 각 팀원과의 토론을 구분해서 보여주세요 — 최종 결론은 [핵심 결론 / 실행 방안 / 남은 리스크] 로 정리해주세요 시작하세요.
핵심: "순서대로 발언"과 "팀장이 결론" 이 두 가지 규칙이 퀄리티를 만듭니다. 규칙 없이 "토론해줘"라고만 하면 산만해집니다.

📝 실제로 이렇게 채우면 됩니다

완성 예시 : 마케팅 전략 수립
지금부터 팀 회의를 시작합니다. ## 논의 주제 20대 직장인 대상 AI 생산성 앱의 런칭 마케팅 전략 ## 팀 구성 (총 4명) 팀장: 마케팅 디렉터 — 각 팀원과 비판적으로 토론하고, 최종 결론을 도출하는 역할 팀원 1: 데이터 분석가 — 시장 규모, 경쟁사 수치, 타겟 사용자 데이터 기반 의견 팀원 2: 고객 경험 전문가 — 20대 직장인의 실제 불편함과 니즈 관점 팀원 3: 재무 담당 — 마케팅 예산 대비 ROI, 비용 효율성 검토 ## 진행 규칙 1. 각 팀원이 자기 관점에서 의견을 먼저 제시합니다 2. 팀장이 각 팀원과 1:1로 3차례씩 비판적 토론을 진행합니다 — 1차: 팀장이 약점·빈틈을 지적 → 팀원이 반박·보완 — 2차: 팀장이 재반박 → 팀원이 근거를 추가해 방어 — 3차: 양측이 합의점 도출 또는 쟁점 정리 3. 모든 팀원과의 토론이 끝나면, 팀장이 논의 내용을 종합하여 최종 결론을 도출합니다 ## 출력 형식 — 각 팀원과의 토론을 구분해서 보여주세요 — 최종 결론은 [핵심 결론 / 실행 방안 / 남은 리스크] 로 정리해주세요 시작하세요.
위 예시를 보세요: 빈칸 템플릿과 구조는 똑같고, 주제와 역할만 구체적으로 채웠을 뿐입니다. 여러분도 본인 업무에 맞게 바꿔 넣으면 됩니다.

※ 토론 라운드 수(3회)는 "주장 → 반박 → 합의"의 최소 논증 구조를 기준으로 한 권장값입니다. 주제의 복잡도에 따라 2~5회로 자유롭게 조정할 수 있습니다.

프로젝트 지침에 넣어서 자동 적용하기

매번 복붙할 필요 없이, 프로젝트 안에서 자동으로 멀티 페르소나가 적용됩니다

💎 Pro · Team · Enterprise 플랜에서 사용 가능

Claude의 프로젝트(Projects) 기능을 사용하면, 위 프롬프트를 프로젝트 지침(Project Instructions)에 저장해둘 수 있습니다.
이렇게 하면 해당 프로젝트에서 새 대화를 시작할 때마다 자동으로 멀티 페르소나 토론 방식이 적용됩니다.

1
프로젝트 만들기
claude.ai 접속 → 왼쪽 사이드바에서 + 프로젝트 만들기 클릭 → 프로젝트 이름 입력 (예: "마케팅 전략 토론")
2
프로젝트 지침에 프롬프트 붙여넣기
프로젝트 상단의 프로젝트 지침 설정 클릭 → 위에서 복사한 멀티 페르소나 프롬프트를 그대로 붙여넣기 → 저장
3
새 대화에서 주제만 입력하면 끝!
이제 해당 프로젝트에서 새 대화를 시작하면 프롬프트가 자동 적용됩니다. 매번 긴 프롬프트를 복붙할 필요 없이, "이번 주제는 ○○야. 시작해줘."라고만 쓰면 됩니다.
TIP: 주제별로 프로젝트를 나눠두면 편합니다. 예를 들어 "마케팅 토론", "사업 아이디어 검증", "보고서 검토" 등 용도별로 프로젝트를 만들고, 각각에 맞는 팀 구성을 프로젝트 지침에 저장해두세요.

이런 상황에서 써보세요

어떤 주제든 팀만 잘 구성하면 됩니다

📄 보고서 검토

팀장: 부서장 / 팀원: 전략가, 현장 실무자, 경영진 관점
"총 4명 팀을 구성해줘. 부서장이 팀장, 전략가·현장 실무자·경영진 관점 담당이 팀원이야. 이 보고서를 검토하는데, 부서장이 각 팀원과 3차례 비판적으로 토론한 후 최종 결론을 내줘."

💡 사업 아이디어 검증

팀장: CEO / 팀원: 시장분석가, 기술전문가, 투자심사역
"총 4명 팀을 구성해줘. CEO가 팀장, 시장분석가·기술전문가·투자심사역이 팀원이야. 이 사업 아이디어를 각자 평가하고, CEO가 각 팀원과 3차례 비판적 토론 후 투자 여부를 결론 내줘."

📧 중요 이메일 작성

팀장: 커뮤니케이션 전문가 / 팀원: 수신자 입장, 법무 검토, 브랜드 담당
"총 4명 팀을 구성해줘. 커뮤니케이션 전문가가 팀장, 수신자 입장·법무 담당·브랜드 담당이 팀원이야. 이 이메일 초안을 각자 평가하고, 팀장이 각 팀원과 3차례 토론한 후 최종 이메일을 완성해줘."

🎯 채용 면접 준비

팀장: HR 디렉터 / 팀원: 직무 전문가, 조직문화 담당, 면접자 입장
"총 4명 팀을 구성해줘. HR 디렉터가 팀장, 직무 전문가·조직문화 담당·면접자 입장이 팀원이야. 면접 질문을 각자 제안하고, HR 디렉터가 각 팀원과 3차례 비판적 토론 후 최종 질문 리스트를 확정해줘."
실전 팁
  • 팀원은 2~4명이 적당합니다. 5명 이상이면 토론이 산만해집니다
  • 반드시 "비판자" 역할을 하나 넣으세요. 모두 찬성하면 토론의 의미가 없습니다
  • 팀장은 직접 의견을 내지 않고 종합만 하게 하면 더 객관적인 결론이 나옵니다
  • 토론 후 "팀장이 실행 계획까지 정리해줘"를 추가하면 바로 써먹을 수 있습니다
  • 한 번 좋은 팀 구성을 찾으면, 비슷한 주제에 계속 재활용하세요
⚠️ 이 기법을 쓰면 안 되는 상황
⚡ 빠른 답변이 필요할 때
멀티 페르소나 토론은 시간이 걸립니다. 단순 정보 검색이나 긴급한 초안 작업에는 직접 질문이 훨씬 빠릅니다.
📐 정답이 하나인 작업
수학 계산, 코드 오류 수정, 번역 등 명확한 정답이 있는 작업에서는 토론이 불필요하고 오히려 혼란을 줄 수 있습니다.
🎨 창의적 자유가 필요할 때
소설, 시, 광고 카피 등 창의적 작업에서는 역할 제약이 오히려 AI의 창의성을 제한합니다. 자유롭게 시도하게 두세요.
🔒 민감한 개인 정보 포함 시
여러 역할에 민감한 정보를 반복 노출하면 데이터 관리 리스크가 높아집니다. 개인정보·기밀 데이터는 최소한의 맥락만 제공하세요.
