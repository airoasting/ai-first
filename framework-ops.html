TOOL 1

4P (마케팅 믹스)

사용 빈도 ★★★★★ 효과성 ★★★★☆

Marketing Mix — 마케팅 전략의 4가지 핵심 요소

Product(제품), Price(가격), Place(유통), Promotion(판촉)의 4가지 마케팅 요소를 체계적으로 설계하는 프레임워크입니다. 마케팅 전략의 가장 기본이면서도 강력한 도구로, 신제품 출시부터 기존 제품의 전략 재설계까지 폭넓게 활용됩니다.

핵심 구성 요소
Product — 무엇을 팔 것인가
Price — 얼마에 팔 것인가
Place — 어디서 팔 것인가
Promotion — 어떻게 알릴 것인가
확장 개념
서비스업에서는 7P로 확장합니다: 기존 4P에 People(사람), Process(프로세스), Physical Evidence(물리적 증거)를 추가하여 서비스 특성을 반영합니다.

언제 쓰는가

  • 신제품 출시 전략 수립 시 각 P별 의사결정 프레임으로 활용
  • 마케팅 캠페인 기획 시 4가지 요소의 일관성 점검
  • 경쟁사 마케팅 전략 비교 시 체계적인 비교 기준으로 활용
AI Prompt 
프리미엄 단백질 쉐이크 신제품의 4P 전략을 수립해줘.
타깃은 30-40대 직장인이고, 온라인 D2C 채널 중심이야.
각 P별로 3가지 핵심 전략과 실행 방안을 제시해줘.
포인트: 타깃 고객과 채널을 명시하고, 각 P별로 구체적 전략 수와 실행 수준까지 요청하면 실무에 바로 활용할 수 있는 결과를 얻을 수 있습니다.
TOOL 2

STP 분석

사용 빈도 ★★★★☆ 효과성 ★★★★★

Segmentation, Targeting, Positioning — 마케팅 전략의 핵심

시장을 세분화(Segmentation)하고, 타깃을 선정(Targeting)하고, 포지셔닝을 결정(Positioning)하는 마케팅 전략의 핵심 프레임워크입니다. "누구에게, 어떤 가치를, 어떻게 전달할 것인가"를 체계적으로 설계합니다.

핵심 구성 요소
Segmentation — 시장 세분화 기준 (인구통계, 심리, 행동, 지리적 변수)
Targeting — 타깃 시장 선택 (집중형, 차별형, 비차별형)
Positioning — 차별화 포지셔닝 (경쟁 대비 고유 가치 정의)

언제 쓰는가

  • 브랜드 론칭 시 시장 내 최적 포지션 결정
  • 시장 재포지셔닝 시 현재 위치 진단과 새로운 방향 설정
  • 신규 타깃 시장 발굴 시 미충족 수요 세그먼트 탐색
AI Prompt 
국내 프리미엄 펫푸드 시장의 STP 분석을 해줘.
세분화 기준 4가지를 제안하고,
가장 매력적인 타깃 세그먼트를 선정하고,
경쟁사 대비 차별화 포지셔닝 맵을 그려줘.
포인트: S-T-P 각 단계별로 구체적 아웃풋을 지정하면, AI가 단계별로 구조화된 분석 결과를 제공합니다. 특히 포지셔닝 맵 요청은 시각적 차별화를 돕습니다.
TOOL 3

Customer Journey Map

사용 빈도 ★★★★☆ 효과성 ★★★★☆

고객 여정 지도 — 고객 경험의 전체 흐름을 시각화

고객이 브랜드를 인지하고 구매·사용·재구매까지의 전체 경험을 시각화하는 도구입니다. 각 단계에서 고객이 무엇을 느끼고, 어디서 이탈하는지를 파악하여 경험을 개선하는 데 활용됩니다.

핵심 단계
인지(Awareness) → 탐색(Consideration) → 구매(Purchase) → 사용(Usage) → 충성(Loyalty)

각 단계별 분석 요소: 고객 행동 / 접점(Touchpoint) / 감정 / Pain Point / 개선 기회

언제 쓰는가

  • UX 개선 프로젝트에서 고객 Pain Point 진단
  • 디지털 전환 시 온·오프라인 접점 재설계
  • 고객 이탈 방지를 위한 핵심 이탈 구간 파악
  • 옴니채널 전략 수립 시 채널 간 경험 일관성 점검
AI Prompt 
온라인 패션 쇼핑몰의 Customer Journey Map을 만들어줘.
5단계(인지~충성)별로 고객 행동, 주요 접점, 감정 곡선,
Pain Point, 개선 기회를 정리하고,
가장 이탈이 많은 구간을 진단해줘.
포인트: 5단계 구조를 명시하고 각 단계별 분석 항목을 구체적으로 나열하면, AI가 표 형태로 체계적인 여정 지도를 작성합니다. 이탈 구간 진단 요청은 실행 우선순위를 잡는 데 도움이 됩니다.
TOOL 4

가치 사슬 분석

사용 빈도 ★★★★★ 효과성 ★★★★★

Value Chain Analysis — 기업 내부 활동의 가치 창출 구조 분석

마이클 포터가 제안한 기업 내부 활동 분석 도구입니다. 가치 창출에 기여하는 활동을 체계적으로 분류하여 경쟁 우위의 원천을 파악하고, 비용 절감과 차별화 기회를 발굴합니다.

핵심 구성 요소
본원적 활동: 물류(Inbound) → 생산(Operations) → 출하(Outbound) → 마케팅/영업 → 서비스
지원 활동: 기업 인프라 / HR(인적자원관리) / 기술개발 / 조달(구매)

언제 쓰는가

  • 경쟁 우위 원천 파악 — 어떤 활동이 핵심 가치를 만드는지 진단
  • 아웃소싱 의사결정 — 내재화 vs 외주화 판단 기준 마련
  • 비용 절감 기회 발굴 — 비효율 활동 식별 및 최적화
AI Prompt 
전자상거래 기업의 가치 사슬을 분석해줘.
본원적 활동 5가지와 지원 활동 4가지를 각각 정리하고,
경쟁 우위를 만들 수 있는 핵심 활동 Top 3를 선정하고
그 이유를 설명해줘.
포인트: 본원적 활동과 지원 활동을 구분하여 요청하고, Top 3 선정과 이유까지 요구하면 단순 나열이 아닌 전략적 인사이트를 얻을 수 있습니다.
TOOL 5

맥킨지 7S

사용 빈도 ★★★★☆ 효과성 ★★★★☆

McKinsey 7S Framework — 조직 효과성 진단 도구

조직 효과성을 7가지 요소로 분석하여 전략 실행의 정렬성을 진단하는 도구입니다. 핵심은 7개 요소의 '정렬(Alignment)'이며, 하나만 바꿔도 나머지 요소에 연쇄적으로 영향을 줍니다.

핵심 구성 요소
Hard 3S (상대적으로 변경이 쉬운 요소)
  • Strategy(전략) / Structure(구조) / Systems(시스템)

Soft 4S (변경에 시간이 걸리는 요소)
  • Shared Values(공유 가치) / Skills(역량) / Style(리더십 스타일) / Staff(인재)
핵심 포인트
7개 요소의 '정렬'이 핵심입니다. 전략(Strategy)만 바꿔놓고 구조(Structure)나 시스템(Systems)을 안 바꾸면, 전략은 실행되지 않습니다. 하나를 바꾸면 나머지 6개도 함께 점검해야 합니다.

언제 쓰는가

  • 조직 변혁 시 현재 조직의 정렬 상태 진단
  • M&A 후 통합(PMI) 과정에서 두 조직의 적합성 분석
  • 전략-실행 간 갭 진단 — 왜 좋은 전략이 실행 안 되는지 파악
AI Prompt 
우리 조직(IT 스타트업, 50명 규모)의 맥킨지 7S 분석을 해줘.
현재 전략은 B2B SaaS 확장인데,
7개 요소별로 현재 상태를 진단하고 정렬도를 평가해줘.
가장 시급히 개선할 요소를 추천해줘.
포인트: 조직 규모와 현재 전략 방향을 명시하면, AI가 맥락에 맞는 구체적 진단을 제공합니다. 정렬도 평가와 우선순위 추천까지 요청하면 실행 가능한 인사이트를 얻습니다.
TOOL 6

역량 성숙도 평가

사용 빈도 ★★★☆☆ 효과성 ★★★★☆

Capability Maturity Model — 조직 역량 수준의 단계별 평가

조직의 프로세스나 역량 수준을 단계별로 평가하는 프레임워크입니다. CMM/CMMI에서 유래했으며, IT뿐 아니라 마케팅, 데이터, HR 등 다양한 영역의 성숙도 진단에 범용적으로 활용됩니다.

핵심 5단계
Level 1 초기(Ad-hoc) — 프로세스가 없고, 개인 역량에 의존
Level 2 반복(Repeatable) — 기본 프로세스가 있지만 일부만 반복 가능
Level 3 정의(Defined) — 표준 프로세스가 조직 전체에 적용됨
Level 4 관리(Managed) — 프로세스를 정량적으로 측정·관리
Level 5 최적화(Optimizing) — 지속적 개선이 체계적으로 이루어짐

언제 쓰는가

  • IT·운영 역량 진단 — 현재 수준을 객관적으로 평가
  • 디지털 성숙도 평가 — 디지털 전환의 현재 단계 파악
  • 프로세스 개선 로드맵 수립 — 다음 단계로 가기 위한 액션 도출
AI Prompt 
우리 팀의 데이터 분석 역량 성숙도를 평가해줘.
5단계(초기~최적화) 기준으로 현재 수준을 진단하고,
다음 단계로 올라가기 위해 필요한
구체적 액션 아이템을 5가지 제안해줘.
포인트: 5단계 기준을 명시하고, 현재 진단뿐 아니라 "다음 단계로 올라가기 위한" 액션까지 요청하면, 진단에서 실행까지 이어지는 실용적 결과를 얻을 수 있습니다.
TOOL 7

프로세스 분해

사용 빈도 ★★★★★ 효과성 ★★★★☆

Process Decomposition — 복잡한 업무를 계층적으로 분해

복잡한 업무 프로세스를 단계별로 분해하여 병목과 비효율을 파악하는 기법입니다. 대분류에서 단위 활동까지 계층적으로 쪼개면, 어디서 시간이 걸리고 어디를 자동화할 수 있는지가 보입니다.

핵심 방법
대분류 → 중분류 → 소분류 → 단위 활동으로 계층적 분해

분석 포인트: 각 단계별 소요 시간 / 담당자 / 인풋·아웃풋 / 병목 구간 / 자동화 가능 여부

언제 쓰는가

  • 업무 효율화 — 불필요한 단계 제거, 병목 해소
  • RPA·자동화 대상 선정 — 반복적·규칙적 활동 식별
  • 서비스 설계 — 고객 대면 프로세스의 세부 흐름 정의
AI Prompt 
고객 주문부터 배송까지의 프로세스를 분해해줘.
대분류 5단계로 나누고,
각 단계를 3-5개 세부 활동으로 분해하고,
각 활동의 예상 소요시간과 병목 가능성(상/중/하)을
표로 정리해줘.
포인트: 분해 수준(대분류 5단계, 세부 3-5개)을 구체적으로 지정하고, 소요시간과 병목 가능성까지 표로 요청하면 즉시 실행 가능한 프로세스 맵을 얻을 수 있습니다.
TOOL 8

SIPOC

사용 빈도 ★★★★☆ 효과성 ★★★★☆

Supplier-Input-Process-Output-Customer — 프로세스 구조의 전체 그림

프로세스의 흐름과 이해관계자 간 구조를 한눈에 보여주는 도식입니다. 6시그마에서 유래한 이 도구는 프로젝트 초기에 "우리가 어떤 프로세스를 다루고 있는가"를 명확히 정의하는 데 탁월합니다.

핵심 구성 요소
Supplier(공급자) → Input(투입물) → Process(프로세스) → Output(산출물) → Customer(고객)

각 요소를 정의하면 프로세스의 범위, 관련 이해관계자, 투입-산출 관계가 명확해집니다.

언제 쓰는가

  • 프로세스 범위 정의 — 프로젝트 시작 시 분석 범위 합의
  • 이해관계자 매핑 — 누가 무엇을 제공하고 받는지 명확화
  • 품질 개선 프로젝트 착수 — 6시그마 DMAIC의 Define 단계
AI Prompt 
신규 직원 온보딩 프로세스의 SIPOC 다이어그램을 만들어줘.
각 요소(Supplier, Input, Process, Output, Customer)를
구체적으로 정리하고,
프로세스 개선이 필요한 포인트를 3가지 지적해줘.
포인트: SIPOC의 5가지 요소를 명시적으로 나열하여 요청하고, 개선 포인트까지 함께 요구하면 현황 파악과 개선 방향을 동시에 얻을 수 있습니다.
TOOL 9

RACI 차트

사용 빈도 ★★★★★ 효과성 ★★★★★

Responsible, Accountable, Consulted, Informed — 역할과 책임의 매트릭스

업무별 역할과 책임을 명확히 구분하는 매트릭스입니다. 프로젝트 관리의 필수 도구로, "이건 누가 하는 거야?"라는 질문이 반복된다면 RACI 차트가 필요한 신호입니다.

핵심 구성 요소
R(Responsible) — 실행 책임자. 실제로 업무를 수행하는 사람
A(Accountable) — 최종 승인자. 결과에 대한 최종 책임을 지는 사람
C(Consulted) — 자문 대상. 의사결정 전에 의견을 구하는 사람
I(Informed) — 정보 공유 대상. 결과를 통보받는 사람
RACI 핵심 규칙
• 각 업무에 A(최종 승인자)는 반드시 1명이어야 합니다
• R(실행 책임자)은 1명 이상 지정해야 합니다

  • 책임 충돌 해소 — "그건 네 일이야" 분쟁을 구조적으로 예방
